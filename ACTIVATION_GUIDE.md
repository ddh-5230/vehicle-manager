# 激活码系统使用指南

## 系统概述

本应用使用基于机器指纹的激活码系统，确保软件只能在特定的机器上运行，防止非法传播。

### 工作流程

1. **首次启动软件**
   - 软件自动生成本机指纹（基于 MAC 地址 + 硬盘序列号）
   - 弹窗显示指纹，用户可以复制

2. **获取激活码**
   - 用户把指纹发送给开发者
   - 开发者使用生成工具创建对应的激活码
   - 开发者把激活码发送给用户

3. **激活软件**
   - 用户在软件中输入激活码
   - 系统验证激活码与指纹是否匹配
   - 验证成功后，保存激活状态（后续无需再输入）

## 开发者工具

### 生成激活码

使用生成工具为用户的指纹创建激活码：

```bash
# 进入项目目录
cd vehicle-manager.worktrees/agents-activation-code-with-fingerprint

# 运行生成工具
node main/generate-activation.js <用户指纹>
```

**示例：**

```bash
node main/generate-activation.js A1B2C3D4E5F6G7H8I9J0K1L2M3N4O5P6
```

**输出示例：**

```
✅ 激活码生成成功！

指纹: A1B2C3D4E5F6G7H8I9J0K1L2M3N4O5P6
激活码: A1B2C3D4-ABCDEF1234567890-1700000000

请复制激活码发送给用户
```

### 激活码格式说明

激活码由三部分组成，以 `-` 分隔：

```
A1B2C3D4 - ABCDEF1234567890 - 1700000000
├─ 指纹前8位   ├─ HMAC值前16位  └─ Unix时间戳
```

- **指纹前8位**：用于识别目标机器
- **HMAC签名**：验证激活码真实性和完整性
- **时间戳**：记录激活时间，30天后激活码失效

## 用户操作指南

### 首次启动时

1. 软件启动后会弹出"软件激活"对话框
2. 看到 32 个字符的指纹代码
3. 点击"复制"按钮复制指纹
4. 将指纹发送给软件开发者

### 获得激活码后

1. 在"激活码"输入框中粘贴开发者提供的激活码
2. 点击"激活"按钮
3. 如果激活成功，软件将自动重新加载并进入正常使用界面
4. 后续启动时无需再次激活

## 技术实现细节

### 指纹生成

指纹由以下信息组成：

- **MAC 地址**：使用第一个非本地 IPv4 网卡的 MAC 地址
- **硬盘序列号**：系统盘的硬件序列号

```
MAC地址:硬盘序列号 → SHA256哈希 → 取前32个字符（转大写）
```

不同操作系统使用不同的命令获取硬盘序列号：

- **macOS**：使用 `diskutil` 命令
- **Windows**：使用 `wmic` 命令  
- **Linux**：使用 `lsblk` 命令

### 激活码验证

1. 验证指纹前 8 位是否与当前指纹匹配
2. 验证 HMAC 签名是否有效
3. 验证激活时间是否在 30 天内

### 安全性说明

- 激活密钥存储在 `activation.js` 中
- 生产环境应通过环境变量 `ACTIVATION_SECRET` 配置密钥
- 激活文件存储在用户数据目录中（`activation.json`）
- 激活状态在每次启动时都会验证

## 环境变量配置

对于生产部署，设置自定义的激活密钥：

```bash
export ACTIVATION_SECRET="your-custom-secret-key"
npm start
```

## 常见问题

### Q: 指纹不同？

A: 指纹基于硬件信息生成。以下情况会导致指纹改变：

- 更换网卡
- 更换硬盘
- 系统重新安装（可能改变序列号检测方式）
- 虚拟机中运行（虚拟硬件序列号可能不稳定）

### Q: 激活码过期了？

A: 激活码有 30 天的有效期。如果激活状态丢失，需要重新获取新的激活码。

### Q: 如何重置激活状态？

A: 删除激活文件，位置取决于操作系统：

- **Windows**：`%AppData%/VehicleManager/activation.json`
- **macOS**：`~/Library/Application Support/VehicleManager/activation.json`
- **Linux**：`~/.config/VehicleManager/activation.json`

### Q: 生成工具输出"指纹格式不正确"？

A: 确保输入的指纹恰好是 32 个十六进制字符（0-9, A-F）。
