# 车辆检查管理系统数据库设计

## 1. 设计目标

- 本地离线运行，数据全部存储在 SQLite 文件。
- 支持 44 台左右车辆的长期维护和追溯。
- 支持检查项目独立管理，并按车辆进行绑定。
- 支持到期/逾期提醒、历史记录、统计和导出。

## 2. 核心实体关系

- `vehicles`：车辆主数据
- `inspection_items`：检查项目模板（周期配置）
- `vehicle_items`：车辆与检查项目的绑定关系（每车多个项目）
- `inspection_records`：检查执行记录
- `reminder_logs`：每日弹窗提醒去重日志

关系说明：
- 一辆车可绑定多个检查项目。
- 一个检查项目可被多辆车绑定。
- 一辆车在某个项目上可有多条检查记录。
- 车辆删除时，级联删除其绑定、检查记录和提醒日志。

## 3. 表结构

### 3.1 `vehicles`

- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `vehicle_no` TEXT NOT NULL UNIQUE（车辆编号）
- `plate_no` TEXT NOT NULL UNIQUE（车牌号）
- `vehicle_model` TEXT NOT NULL（车型）
- `owner_name` TEXT NOT NULL（负责人）
- `purchase_date` TEXT（YYYY-MM-DD）
- `note` TEXT
- `created_at` TEXT NOT NULL
- `updated_at` TEXT NOT NULL

### 3.2 `inspection_items`

- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `name` TEXT NOT NULL UNIQUE（项目名）
- `cycle_value` INTEGER NOT NULL（周期值）
- `cycle_unit` TEXT NOT NULL（`day`/`month`/`year`）
- `enabled` INTEGER NOT NULL DEFAULT 1（是否启用）
- `created_at` TEXT NOT NULL
- `updated_at` TEXT NOT NULL

### 3.3 `vehicle_items`

- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `vehicle_id` INTEGER NOT NULL REFERENCES `vehicles(id)` ON DELETE CASCADE
- `item_id` INTEGER NOT NULL REFERENCES `inspection_items(id)` ON DELETE CASCADE
- `last_check_date` TEXT（最近检查日期）
- `next_due_date` TEXT（下次到期日期）
- `created_at` TEXT NOT NULL
- `updated_at` TEXT NOT NULL
- UNIQUE(`vehicle_id`, `item_id`)

### 3.4 `inspection_records`

- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `vehicle_id` INTEGER NOT NULL REFERENCES `vehicles(id)` ON DELETE CASCADE
- `item_id` INTEGER NOT NULL REFERENCES `inspection_items(id)` ON DELETE CASCADE
- `check_date` TEXT NOT NULL（YYYY-MM-DD）
- `result` TEXT NOT NULL（`pass`/`fail`）
- `note` TEXT
- `created_at` TEXT NOT NULL
- `updated_at` TEXT NOT NULL

### 3.5 `reminder_logs`

- `id` INTEGER PRIMARY KEY AUTOINCREMENT
- `vehicle_id` INTEGER NOT NULL REFERENCES `vehicles(id)` ON DELETE CASCADE
- `item_id` INTEGER NOT NULL REFERENCES `inspection_items(id)` ON DELETE CASCADE
- `reminder_date` TEXT NOT NULL（提醒日期，YYYY-MM-DD）
- `status` TEXT NOT NULL（`upcoming`/`overdue`）
- `created_at` TEXT NOT NULL
- UNIQUE(`vehicle_id`, `item_id`, `reminder_date`)

## 4. 关键业务规则映射

1. 新增车辆：自动绑定所有启用检查项目；`next_due_date` 按 `purchase_date + 周期` 计算。
2. 复制车辆：复制车辆的项目绑定关系（不复制检查历史）；新车可修改车牌号等信息。
3. 新增/编辑/删除检查记录：自动重算该车该项目的 `last_check_date` 和 `next_due_date`。
4. 提醒逻辑：
- `next_due_date` 在未来 7 天内：即将到期
- `next_due_date` 小于今天：已逾期
- 通过 `reminder_logs` 保证“每日一次提醒”
- 录入新检查后自动取消后续提醒（清理该项目历史提醒日志）
5. 删除规则：
- 删除车辆：级联删除全部相关数据
- 删除车辆项目：仅删除该车辆与该项目关联数据，不影响其他车辆

## 5. 索引建议

- `inspection_records(vehicle_id, item_id, check_date)`
- `vehicle_items(next_due_date)`
- `reminder_logs(reminder_date)`
