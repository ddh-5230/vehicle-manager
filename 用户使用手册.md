# 车辆检查管理系统操作手册

## 1. 手册说明

本手册面向系统日常使用人员，主要用于指导用户完成车辆资料维护、检查项目维护、检查记录录入、到期提醒查看、条件查询和 Excel 导出等工作。

本手册尽量使用实际操作视角进行说明，不涉及开发技术细节，适合直接发给业务使用人员阅读。

## 2. 系统用途

车辆检查管理系统用于统一管理危险品运输车辆的检查工作，帮助企业完成以下事项：

- 管理车辆基础信息
- 管理车辆对应的检查项目
- 记录每次检查结果
- 自动计算下次到期时间
- 查看即将到期和已逾期项目
- 按条件查询历史记录
- 导出检查数据和到期数据

## 3. 适用人员

本系统适合以下人员使用：

- 车辆管理员
- 安全管理人员
- 检查记录录入人员
- 负责档案整理和导出报表的工作人员

## 4. 系统主界面简介

系统主界面主要分为两部分：

- 左侧导航栏
- 右侧功能区

### 4.1 左侧导航栏

左侧导航栏用于切换功能页面，包含以下模块：

- `首页总览`
- `车辆管理`
- `检查项目`
- `检查记录`
- `查询导出`

### 4.2 右侧功能区

右侧功能区会随着左侧菜单切换，显示对应模块内容。

页面顶部通常会显示：

- 当前页面标题
- 当前日期
- 刷新按钮

## 5. 快速上手流程

第一次正式使用，建议按下面顺序完成：

1. 先进入 `检查项目`，确认系统中的检查项目和周期是否正确
2. 再进入 `车辆管理`，录入车辆资料或通过 Excel 批量导入
3. 检查每辆车是否已经绑定正确的检查项目
4. 当有实际检查发生时，进入 `检查记录` 录入检查结果
5. 每天优先查看 `首页总览`，关注即将到期和已逾期项目
6. 需要整理报表时，进入 `查询导出` 筛选并导出 Excel

## 6. 首页总览操作说明

首页总览用于快速查看当前整体情况，是建议每天首先查看的页面。

### 6.1 首页包含的内容

首页通常包含以下信息：

- 汇总统计卡片
- 已逾期车辆项目列表
- 7 天内到期项目列表

### 6.2 汇总统计卡片

统计卡片用于快速了解当前数据概况，例如：

- 车辆总数
- 项目总数
- 本月检查数量
- 即将到期数量
- 已逾期数量

不同版本展示的卡片内容可能略有差异，但作用都是帮助使用人员快速掌握当前整体情况。

### 6.3 已逾期车辆项目

这里会显示已经超过应检查日期但尚未处理的项目。

建议重点关注：

- 车辆编号
- 车牌号
- 项目名称
- 到期日期
- 逾期天数

### 6.4 7 天内到期项目

这里会显示未来 7 天内即将到期的项目，用于提前安排检查工作。

建议在这里提前规划：

- 近期需要检查的车辆
- 检查顺序
- 证照办理或年审安排

### 6.5 刷新数据

如果刚录入了检查记录或修改了车辆信息，首页未立即变化，可以点击页面右上角的 `刷新数据` 按钮重新加载。

## 7. 车辆管理操作说明

车辆管理页面用于维护所有车辆基础信息。

### 7.1 车辆列表说明

在车辆列表中，通常可以看到以下字段：

- 车辆编号
- 车牌号
- 车型
- 负责人
- 负责人电话
- 购买日期
- 项目数
- 即将到期数量
- 已逾期数量

这些字段可以帮助你快速判断某辆车当前是否存在待处理项目。

### 7.2 新增车辆

#### 操作步骤

1. 进入 `车辆管理`
2. 点击 `新增车辆`
3. 按要求填写车辆信息
4. 点击保存

#### 建议填写的信息

- 车辆编号
- 车牌号
- 车型
- 负责人
- 负责人电话
- 购买日期
- 备注

#### 使用建议

- 车辆编号建议保持统一格式，便于后续查询
- 车牌号应录入完整，避免后续重复
- 购买日期尽量准确，因为部分项目的初始到期时间可能会参考该日期

### 7.3 编辑车辆

当车辆资料有变化时，可以编辑原有车辆信息。

#### 适用场景

- 负责人变更
- 联系电话变更
- 备注更新
- 车型信息修正

#### 操作步骤

1. 在车辆列表中找到目标车辆
2. 点击对应的 `编辑`
3. 修改信息
4. 点击保存

### 7.4 复制车辆

复制车辆功能适合用于新增“配置相近”的车辆。

例如：

- 新购入同类型车辆
- 大部分检查项目一致
- 只需要调整车牌号、车辆编号等少量信息

#### 操作步骤

1. 在车辆列表中找到一辆配置相近的车辆
2. 点击 `复制`
3. 修改新车辆的基础信息
4. 保存

#### 复制后的效果

- 会复制原车辆的项目配置
- 不会复制原车辆的历史检查记录

### 7.5 删除车辆

如果某辆车不再纳入管理，可以删除。

#### 删除前请确认

- 是否还需要保留该车历史记录
- 是否已经导出过相关数据
- 是否确认不再需要在系统中追踪该车

#### 操作步骤

1. 在车辆列表中找到目标车辆
2. 点击 `删除`
3. 按提示确认

### 7.6 查看车辆项目

车辆列表通常支持查看某辆车已绑定的检查项目。

点击后可以查看：

- 项目名称
- 周期
- 最近检查时间
- 下次到期时间
- 当前状态

### 7.7 关联项目到车辆

如果某辆车需要新增某个检查项目，可以手动关联。

#### 操作步骤

1. 先在车辆列表中选中或查看某辆车
2. 找到车辆项目区域
3. 点击 `关联项目`
4. 选择需要绑定的项目
5. 保存

## 8. 检查项目管理操作说明

检查项目管理用于统一维护系统中的检查项目。

### 8.1 检查项目包含的信息

每个检查项目通常包含：

- 项目名称
- 周期值
- 周期单位
- 是否启用

### 8.2 新增检查项目

#### 操作步骤

1. 进入 `检查项目`
2. 点击 `新增项目`
3. 输入项目名称
4. 设置周期值
5. 选择周期单位
6. 设置启用状态
7. 保存

#### 示例

- 项目名称：`车辆年审`
- 周期值：`1`
- 周期单位：`年`

### 8.3 编辑检查项目

当检查周期或名称需要调整时，可以编辑已有项目。

#### 常见场景

- 周期值变化
- 项目名称调整
- 临时停用某个项目

#### 操作步骤

1. 找到目标项目
2. 点击 `编辑`
3. 修改后保存

### 8.4 启用和停用项目

如果某个项目暂时不需要使用，建议优先停用，而不是直接删除。

#### 停用适合的场景

- 当前阶段暂不适用
- 未来可能恢复使用
- 想保留原有项目定义

### 8.5 删除检查项目

删除项目前请确认：

- 是否还有车辆正在使用该项目
- 是否会影响当前管理工作

建议只有在确定不再使用时才删除。

## 9. 检查记录操作说明

检查记录页面用于录入每一次实际检查情况。

### 9.1 录入检查记录

#### 操作步骤

1. 进入 `检查记录`
2. 选择车辆
3. 选择项目
4. 选择检查日期
5. 选择检查结果
6. 填写备注
7. 点击 `保存记录`

### 9.2 检查结果填写建议

一般可以选择：

- `合格`
- `不合格`

如现场情况复杂，建议在备注中补充说明。

### 9.3 备注内容建议

以下情况建议填写备注：

- 发现异常但已处理
- 发现异常需后续整改
- 临时特殊情况说明
- 补录历史数据说明

### 9.4 保存后系统的自动处理

录入检查记录后，系统会自动：

- 更新该项目最近检查时间
- 重新计算下次到期时间
- 更新提醒状态

### 9.5 修改检查记录

如果检查日期、结果或备注录入错误，可以修改对应记录。

#### 操作步骤

1. 在历史记录列表中找到目标记录
2. 点击 `编辑`
3. 修改内容
4. 保存

修改后，系统会按最新信息重新计算后续到期时间。

### 9.6 删除检查记录

如果某条记录是误录，可以删除。

#### 删除前建议确认

- 是否确实录错
- 删除后是否会影响下次到期时间
- 是否需要保留历史痕迹

## 10. 查询导出操作说明

查询导出页面用于筛选系统数据，并将结果导出到 Excel。

### 10.1 可查询的常见内容

用户通常可以按以下维度查询：

- 车辆
- 项目
- 时间范围
- 即将到期
- 本月需检查
- 已逾期

### 10.2 查询步骤

1. 进入 `查询导出`
2. 选择查询条件
3. 点击查询
4. 查看结果表格

### 10.3 常见查询场景

#### 查询 7 天内到期项目

适合提前安排近期工作。

#### 查询本月需检查项目

适合按月份汇总工作计划。

#### 查询已逾期项目

适合集中处理未完成事项。

#### 按车辆筛选

适合查看单台车的检查历史和到期情况。

#### 按项目筛选

适合查看某一类检查项目的整体情况。

### 10.4 导出 Excel

在查询结果页中，通常可以直接导出当前结果。

#### 操作步骤

1. 先完成查询
2. 点击导出按钮
3. 选择保存位置
4. 保存 Excel 文件

### 10.5 导出前建议

- 先确认筛选条件正确
- 文件名建议按日期或用途命名
- 重要报表建议及时备份

## 11. Excel 批量导入车辆

如果车辆较多，可以使用 Excel 批量导入。

### 11.1 适用场景

- 系统首次建档
- 一次性新增多辆车
- 已有车辆资料已整理在 Excel 中

### 11.2 操作步骤

1. 进入 `车辆管理`
2. 点击 `Excel批量导入`
3. 选择 Excel 文件
4. 查看导入预览
5. 确认后执行导入

### 11.3 导入预览时需要重点检查

- 车辆编号是否完整
- 车牌号是否完整
- 负责人信息是否正确
- 是否存在重复数据
- 是否有空白行或异常行

### 11.4 导入建议

- 导入前先备份原始 Excel
- 首次导入时建议少量测试
- 确认预览无误后再正式导入

## 12. 提醒与状态说明

系统会自动根据检查周期判断项目状态。

### 12.1 常见状态

- `正常`
- `7天内到期`
- `已逾期`

### 12.2 7 天内到期

表示该项目距离到期时间已经比较近，建议尽快安排检查。

### 12.3 已逾期

表示该项目已经超过应完成时间，建议优先处理。

### 12.4 为什么状态会变化

项目状态会受到以下因素影响：

- 购买日期
- 最近检查日期
- 项目周期
- 当前日期

### 12.5 录入新检查记录后会怎样

录入新记录后，系统会重新计算该项目的下一次到期时间，因此状态也会同步更新。

## 13. 典型工作场景操作示例

### 场景 1：新增一台新车并开始管理

操作建议：

1. 进入 `车辆管理`
2. 点击 `新增车辆`
3. 填写车辆资料
4. 保存
5. 查看该车项目配置是否完整

### 场景 2：新车和旧车配置相近

操作建议：

1. 选择一台配置最接近的旧车
2. 点击 `复制`
3. 修改车牌号、车辆编号等信息
4. 保存

### 场景 3：今天完成了一项车辆检查

操作建议：

1. 进入 `检查记录`
2. 选择车辆和项目
3. 录入今天的检查日期
4. 填写结果和备注
5. 保存

### 场景 4：想知道这个月有哪些车要检查

操作建议：

1. 进入 `查询导出`
2. 使用本月相关条件进行筛选
3. 查看结果
4. 如需报表，直接导出 Excel

### 场景 5：发现某辆车已经逾期

操作建议：

1. 在首页查看逾期项目
2. 确认对应车辆和项目
3. 尽快安排检查
4. 检查完成后立即录入记录

## 14. 使用规范建议

为了保证数据准确，建议按以下规范使用系统：

- 车辆编号命名方式保持统一
- 车牌号录入时避免多空格和错别字
- 检查完成后当天或尽快录入
- 修改和删除数据前先确认
- 导出重要数据留档备份

## 15. 常见问题

### 15.1 为什么首页和查询结果不一致

可能原因包括：

- 页面尚未刷新
- 查询条件设置不同
- 刚录入或修改的数据尚未重新加载

建议先点击 `刷新数据` 后再核对。

### 15.2 为什么某辆车没有显示项目

可能原因包括：

- 尚未给该车关联项目
- 项目被停用
- 当前查看的不是该车辆对应详情

### 15.3 为什么录入记录后到期时间变了

这是正常现象。系统会根据最新检查记录自动重新计算下次到期时间。

### 15.4 删除后还能恢复吗

通常不能直接恢复，因此删除前建议再次确认。

### 15.5 为什么导出的数据和预期不一样

通常需要先检查：

- 查询条件是否正确
- 时间范围是否正确
- 是否选择了错误的车辆或项目

## 16. 日常使用建议

建议养成以下习惯：

- 每天先看首页总览
- 重点先处理已逾期项目
- 每次检查后及时录入记录
- 每周或每月导出一次关键数据
- 重要调整前先确认并备份

## 17. 建议的管理流程

推荐日常流程如下：

1. 查看首页待办情况
2. 安排近期检查任务
3. 完成实际检查
4. 录入检查记录
5. 查询和导出月度数据
6. 定期核对车辆资料与项目配置
