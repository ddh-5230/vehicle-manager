## 📋 完整工作流程

### 用户端（首次启动）

1. **启动应用**
   ```bash
   npm start
   ```

2. **看到激活对话框**
   - 显示 32 字符的指纹代码
   - 可点击"复制"按钮复制指纹

3. **发送指纹给开发者**
   - 将指纹粘贴到邮件或消息

### 开发者端

1. **接收用户指纹**
   ```
   用户发送：FEA13C16740524ADB92A7BDB8F3E5996
   ```

2. **生成激活码**
   ```bash
   node main/generate-activation.js FEA13C16740524ADB92A7BDB8F3E5996
   ```

3. **发送激活码给用户**
   ```
   发送：FEA13C16-713B56AD1B1E3801-1781015095
   ```

### 用户端（激活）

1. **输入激活码**
   - 在"激活码"输入框中粘贴
   - 点击"激活"按钮

2. **激活完成**
   - 应用自动重载
   - 进入正常使用界面
   - 后续启动无需再激活

## 🔧 关键文件

## ⚙️ 环境配置

### 生产环境
```bash
# 设置自定义激活密钥
export ACTIVATION_SECRET="your-secure-secret-key"
npm start
```

### 默认配置
- 激活密钥：`vehicle-manager-secret-key-2024`
- 激活有效期：30 天
- 指纹长度：32 个十六进制字符
- 激活码格式：`[8位指纹]-[16位签名]-[时间戳]`

## 🐛 故障排除

### 问题：指纹不同
**解决**：指纹基于硬件信息生成
- 更换网卡会改变指纹
- 更换硬盘会改变指纹
- 系统重装可能改变指纹

### 问题：激活码过期
**解决**：激活码有 30 天有效期
- 需要重新生成新激活码
- 删除 `userData/activation.json` 重置激活状态

### 问题：激活失败
**解决**：检查以下事项
- 激活码格式是否正确（应包含两个 `-`）
- 指纹和激活码是否匹配
- 激活码是否在 30 天有效期内
