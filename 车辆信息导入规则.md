# 车辆信息导入规则

## 1. 文件格式

- 支持 `.xlsx`、`.xls` 文件。
- 默认读取第一个工作表（Sheet）。
- 第一行必须为表头。

## 2. 支持的表头

系统支持根据表头自动匹配字段，推荐使用以下名称：

- `车辆编号`（或：`车编号`）
- `车牌号`（或：`车牌`）
- `车型`（或：`车辆类型`）
- `负责人`（或：`责任人`）
- `负责人电话`（或：`联系电话`、`手机号`）
- `购买日期`（或：`购置日期`）
- `备注`（或：`说明`）

## 3. 必填字段

以下字段为必填：

- `车辆编号`
- `车牌号`
- `车型`
- `负责人`

缺失任一必填字段时，该行导入失败，并记录失败原因。

## 4. 重复处理规则

- `车辆编号` 与 `车牌号` 在系统中均为唯一字段。
- 若导入数据与系统现有数据重复，则该行**跳过**，不重复录入。
- 若同一 Excel 内出现重复（车辆编号或车牌号重复），后出现的行会被**跳过**。
- 当前导入策略为：**只新增，不更新已有数据**。

## 5. 数据校验规则

- `负责人电话` 可为空；非空时需符合手机号或座机格式。
- `购买日期` 支持：
  - Excel 日期格式
  - `YYYY-MM-DD` 文本格式
- 日期格式不正确时，该行导入失败。
- 系统会自动去除文本前后空格。

## 6. 导入流程

1. 选择 Excel 文件  
2. 系统自动识别表头并进行预检查  
3. 显示预览统计（可新增、重复跳过、失败条数）  
4. 用户确认后执行导入  
5. 显示导入结果明细  

## 7. 导入结果说明

导入完成后会显示：

- 成功新增条数
- 跳过明细（Excel 行号 + 原因）
- 失败明细（Excel 行号 + 原因）

示例原因包括：

- `车牌号为空`
- `日期格式错误`
- `唯一冲突：车辆编号或车牌号已存在`

## 8. 导入成功后的系统处理

- 成功新增的车辆会自动绑定当前启用的检查项目。
